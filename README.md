# Σύγκριση Classic REST API και MCP για διασύνδεση με LLMs

Το repository αυτό παρουσιάζει και συγκρίνει δύο διαφορετικές προσεγγίσεις διασύνδεσης LLMs με εξωτερικά εργαλεία:

- **Κλασικά REST API calls**
- **Model Context Protocol (MCP)**

Η έμφαση δίνεται στον τρόπο με τον οποίο ένα LLM ανακαλύπτει, επιλέγει και χρησιμοποιεί εργαλεία, και όχι σε μετρήσεις απόδοσης.

---

## Γενική Περιγραφή

Υλοποιούνται δύο λειτουργικά ισοδύναμοι servers σε on-premise Debian host.

### Classic REST API Server
- Υλοποίηση με **FastAPI**
- Εκθέτει ρητά HTTP endpoints για:
  - χρήση CPU
  - χρήση μνήμης (RAM)
  - χρήση δίσκου
  - uptime συστήματος
  - πληροφορίες διεργασιών
- Ο client απαιτεί:
  - χειροκίνητο parsing του ερωτήματος
  - αντιστοίχιση intents σε συγκεκριμένα endpoints
  - σύνθεση των JSON απαντήσεων

### MCP Server
- Υλοποίηση με **FastMCP**
- Παρέχει την ίδια λειτουργικότητα ως **MCP tools**
- Τα εργαλεία:
  - περιγράφονται μέσω docstrings
  - ανακαλύπτονται δυναμικά από τον client
- Χρήση transport **SSE (Server-Sent Events)**
- Δεν απαιτείται εκ των προτέρων γνώση των διαθέσιμων εργαλείων

Και οι δύο servers χρησιμοποιούν κοινό module (`system_stats.py`) για το business logic.

---

## LLM Clients

Οι LLM clients υλοποιούνται σε **Google Colab** με χρήση **Gemini**:

- **Classic API client**
  - χειροκίνητη αναγνώριση intents
  - ρητές κλήσεις REST API
  - μετατροπή των JSON αποτελεσμάτων σε φυσική γλώσσα

- **MCP client / agentic χρήση**
  - δυναμική ανακάλυψη εργαλείων (`list_tools`)
  - αυτόματη επιλογή και εκτέλεση tools
  - απουσία hard-coded routing λογικής

---

## Dify Agent

Περιλαμβάνεται αρχείο **Dify YAML**, το οποίο παρουσιάζει agent που:

- συνδέεται σε MCP server
- δεν γνωρίζει εκ των προτέρων τα διαθέσιμα εργαλεία
- βασίζεται αποκλειστικά σε MCP tool calls για την απάντησή του

---

## Συμπεράσματα

Το repository αναδεικνύει ότι το MCP:

- απλοποιεί σημαντικά τον client κώδικα
- υποστηρίζει πιο φυσική agentic συμπεριφορά
- μεταφέρει τη γνώση των εργαλείων στον server
- είναι πιο κατάλληλο για LLM-centric αρχιτεκτονικές

---

## Σημειώσεις

- Θέματα ασφάλειας και authentication δεν εξετάζονται
- Δεν περιλαμβάνονται benchmarks απόδοσης
- Το επίκεντρο είναι η **μεθοδολογία διασύνδεσης και η εμπειρία ανάπτυξης**
